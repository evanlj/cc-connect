# 2026-03-21 Squad 审核控制卡说明（飞书）

## 背景

在群聊中手动输入 `/squad ...` 命令容易漏参，尤其在“审核后用户裁决”流程里需要频繁执行：

- `/squad show-plan`
- `/squad approve-plan`
- `/squad approve-task`
- `/squad judge-review ...`

为降低操作成本，本次新增 **Squad 审核控制卡**，通过按钮直接下发命令。

## 变更摘要

### 1) 菜单新增入口

在飞书快捷菜单卡中新增按钮：

- `Squad 审核控制卡`

点击后会打开专用控制卡。

### 2) 新增控制卡内容

卡片包含输入与操作区：

1. `run_id`（必填）
2. `rework_note`（仅“裁决返工”时必填）
3. 命令按钮：
   - 状态
   - 查看计划
   - 查看任务
   - 运行列表
   - 批准计划
   - 批准任务
   - 裁决通过
   - 裁决返工
4. 模板按钮：
   - 启动模板
   - 计划确认模板
   - 任务确认模板
   - 返工模板

### 3) 卡片回调动作

新增卡片动作类型：

- `squad_cmd`
- `squad_tpl`

动作会被转换成普通文本命令（synthetic command）回注到原消息处理流程，不改动 Squad 主流程实现。

## 使用说明

1. 在飞书群里发送 `菜单`（或点击已有菜单卡）；
2. 点击 `Squad 审核控制卡`；
3. 填写 `run_id`；
4. 按需点击按钮执行命令；
5. 若要返工，先填写 `rework_note`，再点 `裁决返工`。

## 影响与风险

### 影响

- 降低命令输入错误率；
- 提升审核裁决环节可用性；
- 便于群聊场景下快速推进任务。

### 风险

- 未填写 `run_id` 时，按钮无法生成可执行 run 级命令；
- “裁决返工”未填 `rework_note` 时会被拦截并提示。

### 缓解

- 卡片内置明确提示文案；
- 对关键命令做参数缺失校验并返回引导信息。

1. 移除菜单中的 `Squad 审核控制卡` 按钮；
